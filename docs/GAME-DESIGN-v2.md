# 宇宙艦隊遊戲體驗 v2 —「艦橋」：場景制 RPG 架構

**日期**：2026-07-05 ｜ **取代**：proposal/GAME-DESIGN.md（v1 的六幕線性版——保留為提案 demo、不再是產品方向）
**v1 的病（Nelsen 診斷、成立）**：只有文字包裝、一頁全看得到、沒有世界、沒有角色、沒有「想回來」的理由——是會動的簡報、不是遊戲。
**v2 的答案**：不追 3A 畫質、追 **3A 的三件事——世界感、角色感、選擇感**、加上一套誠實的黏著迴圈。

## 一、核心轉向（一句話）

> 從「六幕線性放映」變成「**一個持續存在的地方**」——艦橋是家、艦員是活人、每天有理由回來。

## 二、世界觀與故事鉤（3A 的第一件事：世界感）

- **故事開場**：你繼承了一艘退役的傳奇艦「拓荒者號」。前任艦長只留下一句話：「艦員都還在、他們只是睡著了。把他們一個一個叫醒、這艘艦會記得怎麼飛。」
- **懸念層**（黏著的敘事引擎）：艦上有一間鎖著的「檔案室」——前任艦長的航行日誌。**連續登艦與完成任務會逐頁解鎖**、日誌內容 = 內容經營的心法故事化（前任艦長犯過的錯：追過奇觀流量、學會了共鳴、留下了守則）——把課程的方法論寫成可解鎖的劇情、學員追劇情等於複習課程。
- **語氣**：溫暖帶幽默、不中二；文案品質比照 TEXT-PLAYBOOK 的人話標準。

## 三、五名艦員 = 有性格的 NPC（3A 的第二件事：角色感）

每人：性格設定、口頭禪、對話樹（JSON）、立繪兩表情（平常/高興——Codex 資產）、專屬配色微差與音效主題（WebAudio 合成、每人一組音高）。

| 艦員 | 性格 | 口頭禪（示例、實作時可再修） | 對話樹重點 |
|------|------|------|-----------|
| 情報員（scout） | 話多、好奇心爆棚、什麼都想回報 | 「艦長你一定要看這個——」 | 每日晨會主持人：把真實選題日報包成劇情回報 |
| 守門員（guard） | 嚴謹、毒舌但可靠 | 「這句、AI 腔。重寫。」 | 攔截任務的裁判；玩家跟他吵架會有支線對話 |
| 寫手（writer） | 浪漫、慢熱、對文字有潔癖 | 「等等、這個字的重量不對。」 | 聲線任務線；誇他會害羞 |
| 發行官（publisher） | 急性子、行動派 | 「好了嗎？能發了嗎？現在？」 | 發佈儀式主持；每次真實發佈他最興奮 |
| 分析師（analyst） | 冷靜、只信數字、偶爾冒出冷笑話 | 「數字不會安慰你、但我可以……開玩笑的。」 | 週報解讀；預測遊戲的莊家 |

**對話引擎**（輕量、無框架）：JSON 對話樹（節點：台詞/表情/選項/跳轉/旗標）、選擇寫進 localStorage 旗標、後續對話會**記得你說過什麼**（例：你跟守門員頂過嘴、他下次會提）。這是「選擇感」的最低成本實作、也是與「會動的簡報」的分水嶺。

## 四、場景架構（不再一頁全看到）

**場景制 SPA**：全螢幕一次一場景、場景間黑幕字卡轉場（0.4s、像章節卡）、鏡頭感用視差與緩推（背景星空層/中景艦體層/前景 UI 層、進場時 scale 1.03→1.00 緩推）。

```
開場劇情（首次）→ 艦橋（HUB、家）
                    ├─ 點艦員 → 對話場景（立繪＋對話框＋選項、RPG 式）
                    ├─ 艦內地圖 → 任務板（今日任務/任務鏈）
                    ├─ 檔案室（劇情解鎖頁、連續登艦開鎖）
                    ├─ 觀測窗（成就牆＋航程統計、學員的發文里程碑）
                    └─ 艦長室（設定、聲線檔、品牌檔——工具收納進世界）
```

- 艦橋 HUB 恆動：星空漂移、艦員小像各自待機動畫、有新事件的艦員頭上冒橘色驚嘆點（RPG 慣例、引導視線）
- **手機直式優先**（學員日常用手機開）、桌面加寬景

## 五、三段體驗 × 黏著迴圈（產品的完整弧）

**課前（裝備發放、既有 onboarding 掛進世界）**：完成 = 「艦員甦醒」劇情各播一段、不再只是勾選。
**課中（實戰任務鏈）**：課堂八模組 = 八個主線任務、完成觸發對應艦員的劇情對話；「首航」（真實發佈第一篇）是第一章 Boss 任務、全艦慶祝場景。
**課後 30 天（航海圖、黏著主場）**：
- **每日登艦**：晨會場景（情報員回報今日選題——**接真實選題日報資料**、劇情包裝真功能）、每日一個 10 分鐘任務
- **連續日（streak）**：登艦連續 3/7/14/30 天 → 檔案室日誌逐頁解鎖＋艦階晉升（見習→少尉→中尉→少校→艦長）
- **XP = 航程（光年）**：真實動作給 XP（發佈一篇 +100、回覆讀者 +20、看週報 +30）——**XP 綁真實產出、不綁點擊**（誠實黏著、不做點擊糖）
- **危機事件（隨機 mini-quest）**：守門員攔下一篇危險草稿、玩家判斷改或發——判斷練習包成事件
- **週日艦務會議**：分析師講解真實週報＋預測遊戲對答案
- **期末 Boss（第 30 天）**：獨立完成一篇從選題到發佈的完整航行、頒「獨立航行證書」
**畢業後（艦隊聯盟）**：同梯 cohort = 「第 N 艦隊」、艦隊無線電（社群的敘事包裝）、艦隊排行榜（總航程）——課程社群的世界觀化。

## 六、Showoff 層（給有風 demo 時的煙火）

- 開場劇情的星空穿越（Canvas 粒子隧道、10 秒、可跳過）
- 對話場景的立繪進出、表情切換、對話框逐字＋語氣音
- 首航 Boss 完成的全艦慶祝（全屏粒子＋五艦員同框道賀）
- 檔案室翻頁的紙質動畫

## 七、技術路線（仍然無框架、可 Pages 部署）

- 多檔靜態站：`game/index.html` ＋ `game/js/`（scene-manager.js / dialogue-engine.js / save.js / sfx.js）＋ `game/data/`（對話樹 JSON、任務 JSON、日誌 JSON）＋ `game/assets/`
- 存檔：localStorage（profile、旗標、streak、XP）；日期用本地日界線算 streak
- 效能底線：手機 60fps、粒子場景降階（`prefers-reduced-motion` 全尊重）
- 真實資料接點（課後期）：選題日報／週報先用「示範資料模式」、正式版由 plugin 寫本地 JSON 給遊戲讀——**示範模式必須明標「示範資料」**（宣稱=實測紀律的產品版）

## 八之二、對標研究補充（2026-07-05 查證、來源 URL 在研究代理原檔）

- **委託人記憶的「Wondering」查證**：查無「NotebookLM 團隊做的 Wondering」——最接近的是兩個產品的混合印象：**Huxe**（NotebookLM 負責人 Raiza Martin 團隊 2024 底創辦、AI 音訊、2026-05 已關站）與 **wondering.app**（Learn Anything——AI 個人化課程、小份量＋視覺卡片＋互動題＋streak、與 NotebookLM 無關但**產品形態與本設計同路**、可當直接對標）
- **技法採納**：對話樹資料結構借 Yarn Spinner 命名慣例（node/choice/condition、自寫輕量 JSON 版不引外部引擎）；場景敘事借 scrollytelling 圖層原理（視差層＋觸發浮現）但用場景制點擊驅動、不用長捲動
- **黏著紅線（Duolingo 反噬教訓）**：streak 斷了**不懲罰不內疚**（62% 使用者因斷 streak 有罪惡感、34% 焦慮——這是反面教材）；斷線回歸的文案是「守門員說他沒睡、艦一直在」而不是「你失去了 14 天連續紀錄」；解鎖劇情永不因斷 streak 回鎖
