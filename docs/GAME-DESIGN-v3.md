# 裝備發放 v3 — 乾淨重建規格（一條線、一個狀態、一次一步）

**日期**：2026-07-06 ｜ **依據**：零脈絡玩家實測＋狀態機屍檢（工作流 wf_effb3d79）｜ **性質**：砍掉重建、不是修補——舊殼留在 git 歷史。

## 一、唯一目的

這個網頁只做一件事：**課前、讓一個不懂技術的人在 Nelsen 老師陪伴下、30-45 分鐘裝好自己的 AI 團隊。**
不是展示品（experience.html 才是）、不是課後系統（那在課程裡）、不是自由探索世界（已證明是 context 錯亂之源）。

## 二、玩家流程（單線、一次只看一件事）

```
0 字卡 3 秒：「課前裝備發放 — 跟著 Nelsen 老師、把你的 AI 團隊裝上艦」
1 任命書：輸入名字 → 蓋章
2 老師開場（真頭像 assets/mentor-portrait.png、已獲肖像批准）三句：
   「歡迎上艦、艦長○○。」
   「接下來一關一關來、玩完你電腦上就有一支真的 AI 團隊——開課那天帶著它來。」
   「先說老實話：等一下會用到電腦裡的『終端機』。不用怕、每一步都有圖、我全程都在。
    大約 30-45 分鐘、隨時可以休息、進度我記著。」   ← 零脈絡玩家的 one-fix、誠實預告
3 關卡頁 ×5（🔴 一次只顯示當前一關、絕不五卡並排）：
   [老師引導一句] → [三件套：畫面模擬／你做的動作／你應該看到] → [完成按鈕]
   完成 → 該艦員甦醒小卡（頭像＋2-3 句、性格照設定、非完整對話樹）
        → 老師過場一句（每關不同）→ 自動進下一關
4 全通關 → 全艦甦醒慶祝（唯一一條慶祝路徑）→ 老師收尾＋「裝備完成證書」（含名字）
5 回訪：老師「上次到第 X 關、繼續？」→ 直接進該關
```

頂部常駐一條（唯一的固定元素）：`老師頭像縮圖｜第 X／5 關・關名｜[繼續]`。沒有第二條橫幅（體驗版 banner 移除——這已是正式課前工具、不是 demo）。

## 四、技術鐵律（屍檢結論的反面、逐條落實）

1. **單一狀態源**：一個 localStorage 鍵 `fleetOnboard.v2`＝`{name, currentQuest, done[], seenOpening}`——不再兩店各表
2. **帶鎖轉場**：場景管理器持 transition token、轉場進行中禁止二次轉場並鎖輸入（連點不壞）
3. **計時器歸場景**：每場景持有自己的 timers 陣列、teardown 統一清——離場零殘留計時器
4. **DOM 單一職責**：過場字卡與任命書是兩塊 DOM、不共用；甦醒小卡有自己的容器
5. **單檔邏輯**：`js/app.js` 一檔（目標 <800 行）、`index.html`、`css` 沿用視覺調性裁剪；沿用 `data/install-quests.json`（甦醒台詞可併入此檔精簡版）
6. 保留可用件：SaveSystem 的 API 形狀（改單鍵）、三件套 HTML 元件樣式、五關內容與通關碼機制、白名單連結、`prefers-reduced-motion`

## 五、絕不重犯（評審團實錄、每條附出處）

- ❌ 五張卡並排的牆（小白：「從玩遊戲突然變成請先變成工程師」）→ 一次一關
- ❌ 九成畫面是鎖住的內容（小白：「整個畫面九成碰不到」）→ 畫面上只有現在能做的事
- ❌ 無鎖轉場疊加（屍檢②-1）→ token 鎖
- ❌ 跨場景殘留計時器（屍檢②-2）→ teardown 契約
- ❌ 兩個狀態源對同一畫面各自表述（屍檢③）→ 單鍵
- ❌ 同一 DOM 兼兩種敘事身分（屍檢④ orders 場景）→ 單一職責
- ❌ 安慰劑按鈕（小白：「打勾但寫不給航程、不知道意義何在」）→ 沒有意義的互動不放
