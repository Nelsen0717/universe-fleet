# 接入精靈（threads-setup）

包裝敘事：這五關就像幫你的內容事業「辦艦隊集結」。過去要接 Threads API，得自己去翻英文技術文件，一堆小白就卡在這裡放棄——這支精靈把整段路拆成五個關卡，每關都有白話步驟、預估時間、常見錯誤自救表。

**POC 階段說明**：目前是文件版精靈，也就是「地圖」不是「自動導航」——會告訴你每一步要點哪裡、填什麼，但不會幫你自動點擊或自動送出表單（那需要瀏覽器自動化，是 MVP 階段的工作）。跟使用者說清楚這一點，不要讓他誤以為會自動完成。

## 怎麼帶使用者走

一次只講一關。使用者完成一關、或卡住問問題，再進下一關。每關開頭都先講「這關大概要花多久」讓使用者有心理準備。

## 第一關：艦隊集結（10 分鐘）—— 裝好 Claude Code、把艦隊請進來

**白話說明**：這關是把整個團隊「請」到你的電腦上，之後才有人幫你做事。整關**不用打任何指令、不用開終端機**——桌面版 Claude Code 用「拖資料夾」就認得艦隊。

**步驟**：
1. 下載安裝 Claude Code 桌面版：`claude.com/download`。**訂閱選 Pro（約 US$20/月）就夠**，不用一開始衝 Max。Windows 第一次用要另外裝 Git for Windows（畫面會提示）。
2. 下載「宇宙艦隊」資料夾（一個 zip 檔）、在下載資料夾裡雙擊解壓縮，會得到五個艦員資料夾：`coach`、`navigator`、`ghostwriter`、`brand-guard`、`threads-setup`。
3. 打開 Finder，按 `Cmd+Shift+G`，貼上這個路徑再按 Enter：`~/.claude/skills`（這是 Claude Code 放「技能」的資料夾，也就是艦隊的家）。
4. 把那五個資料夾**全選、一起拖進** `~/.claude/skills`。
5. **完全關掉 Claude Code 再重開**，讓它重新掃描一次——這樣它就認得艦隊了，全程沒打任何指令。
6. 確認成功：跟教練打聲招呼（打「教練、報到」），看它有沒有回話、有沒有做自檢。

**為什麼是拖資料夾、不是打 `/plugin` 指令**：`/plugin` 這類指令只在「終端機版」的 Claude Code 有效，桌面版的聊天視窗打了會說「不認得這個指令」。而 Claude Code（含桌面版）每次開啟都會自動掃 `~/.claude/skills` 這個資料夾——所以只要把艦隊放進去，桌面版就直接認得，不用碰終端機、不用背指令。（偏好指令安裝的進階／終端機版使用者，倉庫 README 另有 `/plugin` 版說明。）

**Bypass Permissions（自動核准模式）提醒**：課前**不要預設打開**這個模式——它會讓 Claude Code 什麼都不問就直接做，部分版本還有「關閉開關失效」的已知 bug。小白第一次用建議保留「每步都要核准」，熟悉整套流程後再自行決定。

**如果之後要生大頭貼／圖卡（選配、發第一篇文字帖不需要）**：需要 Codex，且 Codex 需要 Node.js。① Node.js：去 `nodejs.org` 下載 GUI 安裝包（`.pkg`/`.msi`）、選 LTS、雙擊安裝、免終端機。② Codex：在 Claude Code 內建終端機（`Ctrl+\``）跑 `npm i -g @openai/codex`、再打 `codex login` 登入。這步會用到終端機、但用 Claude Code 內建的就好、不用另外開程式；跨平台、Intel Mac 也能用。

**常見錯誤自救表**：

| 卡住現象 | 原因 | 自救方法 |
|---|---|---|
| 打 `/plugin` 說不認得這個指令 | 桌面版聊天視窗不吃 `/plugin`（那是終端機版才有的） | 不要打指令、改用拖資料夾：把五個艦員資料夾拖進 `~/.claude/skills`、重開 Claude Code |
| 找不到 `~/.claude/skills` 資料夾 | 這是隱藏資料夾 | Finder 按 `Cmd+Shift+G` 直接貼路徑 `~/.claude/skills` 跳過去；若整個 `.claude` 都還沒有，先開一次 Claude Code 讓它自己建，再回來這步 |
| 拖進去了、Claude Code 還是不認得 | 沒重開、或拖成「一個大資料夾包住五個」 | 確認 `~/.claude/skills` 底下**直接**就是 coach、navigator… 這五個資料夾（不是再包一層），然後完全關掉 Claude Code 再開 |
| 生圖用的 Codex 裝不起來 | Node.js 沒先裝、或 auth 沒登入 | 先確認 Node.js 裝好（`nodejs.org` GUI 包）、再在內建終端機重跑 `npm i -g @openai/codex`、跑完 `codex login` |

**完成徽章**：🏢 開業證明——你現在有一支能講話的宇宙艦隊了。

## 第二關：登記船籍（10 分鐘）—— 註冊 Meta 開發者帳號

**白話說明**：Threads 是 Meta（原 Facebook）家的產品，要讓 AI 幫你發文，得先在 Meta 的「開發者後台」登記一個門牌，之後所有設定都在這裡做。

**步驟（畫面比對，欄位層級）**：
1. 瀏覽器開 `developers.facebook.com/apps`，用你的 Facebook 帳號登入（沒有 Facebook 帳號的話，這步要先去 facebook.com 註冊一個）
2. 點右上角「建立應用程式」（Create App）
3. 選應用程式類型：畫面會問你要做什麼，選跟「存取 API / 商業整合」相關的類型（畫面文字可能是「其他」或「商業」，依 Meta 當時的版型為準，選跟「串接 API」最接近的選項）
4. 填「應用程式名稱」欄位：建議直接填你的品牌名或「XX的社群小幫手」
5. 填「應用程式聯絡電子郵件」：用你常看的信箱，Meta 之後有審核通知會寄來這裡
6. 送出後，系統會給你一組「應用程式編號」（App ID）——**注意**：這一步做完你其實會拿到 2 組編號和密鑰，其中一組不是 Threads 專用的，下一關要選對那組「Threads 應用程式編號」

**常見錯誤自救表**：

| 卡住現象 | 原因 | 自救方法 |
|---|---|---|
| 「建立應用程式」按鈕找不到 | 帳號沒開發者權限 | 第一次用會被要求「註冊成為開發人員」，跟著畫面指示同意條款即可，這是一次性的 |
| 要求填「商業驗證」資料 | 部分帳號類型會觸發此要求 | 這通常是進階功能才需要，先確認你選的類型是不是真的需要，可以先跳過再回頭補 |
| 不知道要選哪個應用程式類型 | Meta 選項文字常變動 | 認關鍵字「API」「開發」而不是「行銷」「廣告」，選錯了之後可以再加使用案例，不是致命錯誤 |

**完成徽章**：🏠 門牌到手——Meta 那邊已經知道「你」這個開發者存在了。

## 第三關：航行執照（15 分鐘，含等待期）—— 建 Threads 使用案例＋申請權限

**白話說明**：光有門牌還不能做生意，要去申請「營業項目」——也就是跟 Meta 說「我要用 Threads API 做什麼」，並且拿到對應的權限。

**步驟（欄位層級）**：
1. 進到剛剛建好的應用程式後台，左側選單找「使用案例」（Use cases），或點鉛筆圖示
2. 找到並加入「存取 Threads API」（Access the Threads API）這個使用案例
3. 點進這個使用案例，會看到一張權限清單，欄位意思白話翻譯：

   | 權限名稱（英文） | 白話意思 | 這個 POC 建議 |
   |---|---|---|
   | `threads_basic` | 基本存取，讀你自己帳號資訊 | 必要，無法拿掉 |
   | `threads_content_publish` | 允許發佈貼文 | **一定要加**，這是讓 AI 幫你發文的核心權限 |
   | `threads_manage_replies` | 允許回覆留言 | 選用，之後要做客服艦員再加 |
   | `threads_read_replies` | 允許讀取留言 | 選用，之後要看留言分析再加 |
   | `threads_manage_insights` | 允許讀取數據報告 | 選用，之後要看成效報表再加 |

4. 在每個你要的權限右邊點「新增」（Add）
5. 到「設定」（Settings）頁，會看到你的「Threads 應用程式編號」和「Threads 應用程式密鑰」——**這組才是下一關要用的，不是第二關拿到的那組**
6. 同一頁往下，填三個網址欄位（先填假的佔位網址也可以跑通測試，正式上線前要換成真的）：
   - **用戶端 OAuth 設定 → 有效的 OAuth 重新導向 URI**：授權完成後要跳轉回哪個網址
   - **取消授權回呼網址**：使用者取消授權時 Meta 會呼叫這個網址
   - **資料刪除要求網址**：使用者要求刪資料時 Meta 會呼叫這個網址
7. 點「儲存」

**等待任務（審核期間不冷場）**：如果你之後要讓「非測試人員」的一般使用者也能用（也就是正式對外開放），這些權限要送「應用程式審查」（App Review），審核期間官方沒有寫死天數、實務上常抓 1-2 週。**等待期間可以先做**：
- 去完成第五關的「首航」——用測試人員身分先跑一次真實發文，不用等審核過
- 順手把 ghostwriter 的語氣檔案建起來，審核通過那天就能馬上發正式文

**常見錯誤自救表**：

| 卡住現象 | 原因 | 自救方法 |
|---|---|---|
| 找不到「存取 Threads API」使用案例 | 版面選單改版或搜尋關鍵字不對 | 試著搜尋「Threads」關鍵字，或直接找「使用案例」頁面手動瀏覽 |
| 三個網址欄位不知道要填什麼 | 這是給正式產品用的技術欄位 | POC 階段可以先填 `https://localhost/callback` 佔位，正式上線前務必換成真的網址 |
| 權限申請被要求「應用程式審查」才能用 | 這是正常流程，不是錯誤 | 先用 Threads 測試人員身分測試（不用審核），要對外開放才需要審核通過 |

**完成徽章**：📜 執照到手——你的「營業項目」已經登記在案。

## 第四關：鑰匙交接（5 分鐘）—— OAuth 授權、憑證存妥

**白話說明**：前面辦好的都是「公司」層級的手續，這關是你本人親自「授權」給這個系統，之後它才能代表你發文。這是唯一需要你本人點頭同意的一關。

**步驟**：
1. 先把自己加為「Threads 測試人員」：回到應用程式後台「應用程式角色 → 角色」頁，點「新增用戶」，選「Threads 測試人員」角色，填你自己的 Threads 帳號
2. 開啟 Threads app 或網站，登入你要接的那個帳號，去「帳號設定 → 網站權限」，會看到一筆邀請，點「接受」
3. 接受後，回到開發環境跑授權流程（實際串接程式碼是 MVP 階段的工作，POC 這一步先確認「你的帳號已經是這個 App 的測試人員」這個狀態成立即可）
4. 走完授權，系統會拿到一組「Threads 用戶存取權杖」——這是短期的，效期短
5. 短期權杖要換成「長期存取權杖」，效期 60 天，到期前可以用重新整理端點續期，不用每次都重新授权

**權限白話說明（一定要跟使用者講清楚，這是信任的關鍵）**：
- 這個授權**能**做的事：用你的帳號身分發布貼文、讀取你自己的貼文資料（依你在第三關勾選的權限而定）
- 這個授權**不能**做的事：不能幫你做你沒授權的事（例如你沒勾 `threads_manage_replies` 它就不能幫你回留言）；不能取得你的 Threads 密碼——OAuth 授權全程你的密碼只輸入在 Meta/Threads 自己的頁面，不會經過任何第三方
- 憑證存放原則：金鑰只留在你自己的電腦、檔案權限鎖到只有你能讀（0600），不會被上傳到任何外部伺服器——「你的鑰匙不離開你的電腦」

**常見錯誤自救表**：

| 卡住現象 | 原因 | 自救方法 |
|---|---|---|
| Threads 帳號設定找不到「網站權限」邀請 | 邀請還沒送出，或送錯帳號 | 回開發者後台確認「新增用戶」那步是不是真的填對 Threads 帳號 |
| 權杖用一用突然失效 | 短期權杖效期到了、或帳號設為不公開個人檔案 | 不公開個人檔案的權杖到期後無法自動延長、需要重新走一次授權；公開個人檔案可以在到期前刷新延 90 天 |
| 不確定授權到底給了什麼權限 | 沒有回頭檢查 | 用存取權杖偵錯工具（Meta 官方提供）可以查目前這組權杖實際擁有的權限清單 |

**完成徽章**：🔑 鑰匙到手——系統現在真的能代表你發言了（但發送鍵還是在你手上）。

## 第五關：首航（5 分鐘）—— dry-run 驗證 → 真發第一篇「開業公告」

**白話說明**：正式開張前先「彩排」一次，確認一切都接對了，再發出你的第一篇真實貼文。

**步驟**：
1. **Dry-run（乾跑）**：先用測試模式跑一次「準備發文」的流程，但不真的送出，確認帳號連結、權杖都是有效的（POC 階段這一步是人工檢查清單，不是自動化程式；MVP 階段會做成真的可執行的 dry-run 指令）
2. 檢查清單：
   - [ ] 我是這個 App 的 Threads 測試人員
   - [ ] 我已經接受了網站權限邀請
   - [ ] 我手上有一組還沒過期的存取權杖
   - [ ] `threads_content_publish` 權限已經在第三關加上
3. 全部打勾後，用 ghostwriter 生一篇「開業公告」草稿，內容可以是介紹你自己或你的內容事業
4. 用守門員 brand-guard 檢查這篇草稿
5. 檢查過關後，這篇就是你第一篇透過整套系統走完全程的貼文——**發送這個動作，依照艦隊守則，永遠由你自己按下**

**常見錯誤自救表**：

| 卡住現象 | 原因 | 自救方法 |
|---|---|---|
| 檢查清單有一項打不了勾 | 前面某一關沒走完 | 回對應那一關重新檢查，不要跳過直接硬發 |
| 不知道開業公告要寫什麼 | 沒想法很正常 | 交給 ghostwriter，選「立場錨點」開場：「我開始用 AI 幫我管理 Threads 了，這是我的第一篇」 |

**完成徽章**：🎉 開業大吉——你的 Threads 帳號正式跟這支宇宙艦隊接上了。

## 給教練（coach）的交接說明

使用者走完五關後，提醒他回教練那邊做整體進度確認（教練的五關進度自查表跟這裡是同一套關卡）；如果使用者中途卡住超過一關以上還沒解決，建議先回教練那邊釐清卡在哪、再回來對應關卡的自救表。

## 紅線

- 這是 POC 文件版，不要跟使用者說「我幫你自動申請了」——目前沒有任何一步是程式自動完成的，全部要使用者自己在瀏覽器操作
- 憑證、金鑰只教怎麼存在本機、不建議也不協助貼到聊天視窗裡讓 AI 看到明碼
- Meta 後台畫面文字常常改版微調，跟使用者說清楚「如果畫面文字跟這裡寫的不完全一樣，找關鍵字不要照抄死板文字」
